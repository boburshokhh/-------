# Деплой AI Test Generator на Ubuntu (edusite)

## Состояние портов на сервере (по проверке)

| Порт  | Статус   | Кто использует        |
|-------|----------|------------------------|
| 80    | **свободен** | — (наш nginx)          |
| 3000  | занят    | frontend (edu_atg)     |
| 8000  | занят    | backend (edu_atg)      |
| 5432  | занят    | postgres               |
| 6379  | занят    | redis                  |
| 9000  | занят    | Minio API              |
| 9001  | занят    | Minio Console          |

Наше приложение публикует один хост-порт — задаётся в `.env` переменной **`HOST_PORT`** (по умолчанию 80). Внутри Docker приложение слушает 3000 только в сети контейнеров — конфликта с host:3000 нет. Если 80 займут, в `.env` укажи свободный порт, например `HOST_PORT=8080`.

---

## Одна команда до рабочего состояния

Из каталога проекта на сервере (`/opt/edu_atg_ai_testgen`):

```bash
(test -f .env || printf '%s\n' 'GEMINI_API_KEY=REPLACE_WITH_YOUR_KEY' 'PORT=3000' 'HOST_PORT=80' 'MAX_FILE_SIZE_MB=10' 'CHUNK_TOKEN_LIMIT=2500' 'CHUNK_OVERLAP_TOKENS=200' > .env) && docker compose up -d --build
```

Если `.env` уже есть (с правильным `GEMINI_API_KEY`) — создаваться не будет, сразу пойдёт сборка и запуск. Иначе после первого запуска отредактируй `.env`: `nano .env` и подставь ключ Gemini, затем `docker compose up -d`.

## Шаги деплоя на сервере

### 1. Склонировать/скопировать проект на сервер

```bash
# Вариант: клонирование (если проект в git)
cd ~
git clone <URL_репозитория> edu_atg_ai_testgen
cd edu_atg_ai_testgen
```

Либо скопировать папку проекта (например через scp/rsync с локальной машины).

### 2. Создать .env на сервере

В корне проекта создать файл `.env`:

```bash
nano .env
```

Содержимое (подставьте свой ключ Gemini; `HOST_PORT` — свободный порт на сервере, 80 или например 8080):

```
GEMINI_API_KEY=ваш_ключ_gemini
PORT=3000
HOST_PORT=80
MAX_FILE_SIZE_MB=10
CHUNK_TOKEN_LIMIT=2500
CHUNK_OVERLAP_TOKENS=200
```

### 3. Запустить контейнеры

```bash
cd ~/edu_atg_ai_testgen   # или ваш путь к проекту
docker compose up -d --build
```

### 4. Проверить

```bash
docker ps
curl -s http://localhost/api/health
```

В браузере: `http://<IP_или_домен_сервера>/` — должна открыться страница приложения.

### 5. Остановка / перезапуск

```bash
docker compose down
docker compose up -d
```

### 6. Обновление из Git и перезапуск (одна команда)

Из каталога проекта на сервере:

```bash
cd /opt/edu_atg_ai_testgen && git pull && docker compose up -d --build
```

Подтянет изменения и пересоберёт/перезапустит контейнеры, чтобы изменения вступили в силу.

## Как тестировать приложение после деплоя

### 1. На сервере (быстрая проверка)

```bash
# Контейнеры запущены
docker ps | grep ai-testgen

# Health API отвечает
curl -s http://localhost/api/health
# Ожидается: {"status":"ok","timestamp":"...","hasApiKey":true}

# Если используете не 80, подставьте свой порт:
curl -s "http://localhost:${HOST_PORT:-80}/api/health"
```

Если `hasApiKey: false` — в `.env` не задан или не подхватился `GEMINI_API_KEY`. Логи: `docker compose logs app --tail 50`.

### 2. В браузере (основной сценарий)

Открой **`http://<IP или домен сервера>/`** (если порт не 80 — добавь `:ПОРТ`, например `http://edusite:8080/`).

| Шаг | Что сделать | Ожидаемый результат |
|-----|----------------|----------------------|
| 1 | Открыть главную | Страница «Генератор тестов», форма загрузки файла |
| 2 | Загрузить PDF или DOCX (до 10 МБ) | Прогресс, затем сообщение об успехе и ссылка на тест |
| 3 | Перейти в список тестов / открыть тест | Список тестов или страница с вопросами |
| 4 | Ответить на вопросы и отправить | Результат: баллы и правильные ответы |

Если загрузка «висит» или ошибка — смотреть логи: `docker compose logs app -f`.

### 3. Проверка API вручную (curl)

```bash
# Базовый URL (подставь свой хост и порт)
BASE="http://localhost"
# или: BASE="http://edusite"
# или: BASE="http://192.168.1.10:8080"

# Список тестов
curl -s "$BASE/api/tests"

# Загрузка файла (с сервера, если есть тестовый PDF)
curl -s -X POST -F "file=@/path/to/document.pdf" "$BASE/api/upload"
```

Успешный ответ от `/api/upload` содержит `"success": true` и `testId`.

### 4. Если что-то не работает

| Проблема | Что проверить |
|----------|----------------|
| Страница не открывается | `docker ps`, порт в `.env` (`HOST_PORT`), фаервол |
| 502 Bad Gateway | `docker compose logs app` — приложение не стартовало или падает |
| «Пустой ответ» / ошибка генерации | Ключ Gemini в `.env`, лимиты API, `docker compose logs app` |
| Загрузка файла не идёт | Размер до 10 МБ, формат PDF/DOCX, `client_max_body_size` в nginx уже 10M |

## Важно

- **Порт 80** — единственный порт, который приложение слушает на хосте. Остальные порты (3000, 9000 и т.д.) уже заняты другими сервисами — наше приложение их не использует.
- Minio на сервере уже настроен; текущая версия AI Test Generator хранит файлы в Docker volume `ai-testgen-data`, не в Minio. При необходимости позже можно добавить интеграцию с Minio.

### Смена порта (если 80 занят)

В `.env` на сервере задай свободный порт, например 8080:

```bash
HOST_PORT=8080
```

Перезапусти: `docker compose up -d`. Приложение будет доступно по `http://<сервер>:8080/`.
