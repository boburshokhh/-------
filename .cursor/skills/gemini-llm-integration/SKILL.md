# Gemini LLM Integration

Этот навык описывает, как безопасно работать с вызовами Gemini API в проекте.

## 1. Строгий JSON
Всегда запрашивайте `responseMimeType: "application/json"`. Это гарантирует, что модель вернет валидный JSON, но **промпт тоже должен требовать JSON**.
Если модель возвращает markdown-блоки, используйте функцию `extractJSON` (из `validator.js`).

## 2. Инструкции (System Prompt)
Общие правила для изменения промпта:
- Четко указывайте структуру массива: `[{...}, {...}]`.
- При добавлении новых типов вопросов (сейчас: `multiple_choice`, `true_false`, `open_ended`), добавляйте их структуру в промпт в качестве примера.
- Не просите LLM "подумать", если вам нужен только JSON, иначе JSON сломается. Вся аргументация должна идти внутрь поля `"explanation": "..."`.

## 3. Обработка ошибок (Retries)
Если LLM упала (например, превышены лимиты, тайм-аут или невалидный JSON), **всегда** используйте экспоненциальный backoff:
```javascript
// Уже реализовано в generator.js, используйте этот паттерн для новых вызовов
if (attempt < config.LLM_MAX_RETRIES) {
    await sleep(1000 * Math.pow(2, attempt - 1));
}
```

## 4. Температура
- Для генерации тестов используйте `temperature: 0.7`.
- Если добавляете функционал точного извлечения фактов (Extract) — используйте `temperature: 0.1` или `0.0`.
