# Docs Writer (Поддержка документации)

Документация проекта находится в директории `docs/` и `README.md`. Ее актуальность критична для новых разработчиков и пользователей.

## 1. Когда обновлять документацию?

- Добавлен или изменен API endpoint -> Обновить `docs/api.md`.
- Добавлена новая таблица в БД или изменен алгоритм обработки LLM -> Обновить `docs/architecture.md`.
- Изменился процесс установки или переменные окружения (`.env`) -> Обновить `README.md`.
- Добавлены новые кнопки или шаги в интерфейсе -> Обновить `docs/user-guide.md`.
- Появились новые специфичные ошибки (например, с Gemini API) -> Обновить `docs/troubleshooting.md`.

## 2. Стиль документации

- **Язык**: Русский.
- **Формат**: Markdown.
- **Тон**: Краткий, технический, без лишней воды.
- **Примеры**: Для API *обязательно* приводить пример JSON ответа.

## 3. Workflow для агента
При завершении крупной задачи, всегда:
1. Выделите время, чтобы просмотреть измененные файлы.
2. Проверьте: "Влияет ли это на API? На UI? На запуск?"
3. Если да, самостоятельно внесите правки в соответствующие Markdown файлы, не дожидаясь просьбы пользователя.
